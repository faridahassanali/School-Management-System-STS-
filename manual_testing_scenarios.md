
# Manual Testing Scenarios — SMS-55 & SMS-56 & SMS-57

This file contains manual step-by-step testing scenarios for UI, CRUD, and system behavior.

---

## Scenario 1 — Verify UI Navigation (SMS-56)
**Goal:** Ensure that all modules open without errors  
**Steps:**
1. Launch the system  
2. Click each main menu module (Students, Buses, Drivers, Supervisors, Alerts)  
3. Observe if pages load correctly  

**Expected Result:** All pages open with correct UI elements  

---

## Scenario 2 — CRUD Operations Testing (SMS-57)
### Create
1. Open Students module  
2. Click Add Student  
3. Enter valid data  
4. Save  

### Read
1. Open Students list  
2. Check if newly added student appears  

### Update
1. Select student  
2. Edit data  
3. Save  

### Delete
1. Select student  
2. Delete  
3. Confirm deletion  

**Expected Result:** All CRUD actions function correctly  

---

## Scenario 3 — Bug Reporting Workflow (SMS-58)
**Steps:**
1. Identify a defect  
2. Capture screenshot  
3. Open Jira  
4. Create bug  
5. Add steps, expected vs actual results, severity  
6. Submit  

**Expected Result:** Bug is recorded and visible to team
