# Manual Testing Scenarios

This document describes manual testing scenarios executed for the
School Transportation System (STS).

---

## Scenario 1 — UI Navigation

### Steps:
1. Launch the application
2. Navigate to Students module
3. Navigate to Drivers module
4. Navigate to Buses module
5. Navigate to Alerts module

### Expected Result:
- All screens load correctly without errors

---

## Scenario 2 — CRUD Operations

### Create:
1. Open Students module
2. Enter valid student data
3. Click Save

### Read:
- Verify student appears in table

### Update:
1. Edit student information
2. Click Save

### Delete:
1. Select student
2. Click Delete
3. Confirm deletion

### Expected Result:
- CRUD operations work correctly

---

## Scenario 3 — Bug Reporting Workflow

### Steps:
1. Identify a system defect
2. Log bug in Jira
3. Fill summary, steps, expected vs actual
4. Track status (To Do → In Progress → Done)

### Expected Result:
- Bug is logged and tracked correctly
